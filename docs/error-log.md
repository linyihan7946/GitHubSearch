## 记录格式模板

复制以下模板新增错误条目：

```markdown
### [简短错误标题]

- **发生次数**：N 次
- **首次日期**：YYYY-MM-DD
- **最近日期**：YYYY-MM-DD
- **错误描述**：简要说明错误现象
- **根本原因**：为什么会重复出现
- **解决方案**：如何彻底避免
- **相关文件**：涉及的文件路径
```

## 示例

### 路径未转义导致文件操作失败

- **发生次数**：3 次
- **首次日期**：2026-07-21
- **最近日期**：2026-07-22
- **错误描述**：Windows 下使用 `C:\Users\...` 格式的路径调用 Bash 工具时报错 "No such file or directory"
- **根本原因**：Bash 工具使用 POSIX 路径格式，Windows 反斜杠和冒号会被转义丢失
- **解决方案**：统一使用 POSIX 格式路径（`/c/Users/...`）或 Git Bash 风格路径
- **相关文件**：涉及文件移动、复制、读取的所有操作
